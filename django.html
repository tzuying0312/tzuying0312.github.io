Background

Django 是一個 Python 網路框架，可以快速開發安全和可維護的網站，類似 Flask，但功能叫 Flask 完善許多。

Install

下載套件

pip install django


建立 Project

django-admin startproject potrip . 建立一個 project 叫 potrip，結構如下。


    ├── manage.py # 用來管理整個 Django 專案，像是啟動本地端伺服器、連接資料庫及建立應用程式(APP)等
    └── potrip
        ├── __init__.py # 告訴 Python potrip是一個套件
        ├── asgi.py # 提供非同步的功能。
        ├── settings.py # Django 專案的設定檔(像 secret key、database、template engine、app 等) 原始設定是 SQLite
        ├── urls.py # 各個應用程式(APP)的網址(就是哪個路徑要用哪個 function 去處理)
        └── wsgi.py # 提供 Django 網站和伺服器間的標準介面(通常不會動這個檔案)

建立 App

python manage.py startapp myapp 建立一個 app 叫 myapp，可以發現在資料夾中有新的檔案，結構如下。


    ├── manage.py
    ├── myapp
    │   ├── __init__.py 
    │   ├── admin.py # 可以用來定義或客製化這個應用程式(APP)在 Django 後台(Django Administration)的欄位顯示方式
    │   ├── apps.py # 應用程式的設定檔
    │   ├── migrations # 記錄資料庫與 models.py 中的欄位同步歷程
    │   │   └── __init__.py
    │   ├── models.py # 定義 Django 應用程式的資料庫欄位
    │   ├── tests.py # 能夠撰寫 Django 應用程式的自動化測試腳本
    │   └── views.py # 負責接收瀏覽器的請求，進行邏輯的處理後，回傳執行結果給瀏覽器
    └── potrip
        ├── __init__.py
        ├── __pycache__
        │   ├── __init__.cpython-37.pyc
        │   └── settings.cpython-37.pyc
        ├── asgi.py
        ├── settings.py
        ├── urls.py
        └── wsgi.py

此時，為了要讓 project 知道有一個 app，因此要在把這個新的 app 加到 project 裡，修改 potrip/settings.py，如下。


    # /potrip/settings.py
    INSTALLED_APPS = (
        'myapp', # myapp.apps.PostsConfig 也可以，讓 potrip 知道有 myapp 的存在
        'django.contrib.admin',
        'django.contrib.auth',
        'django.contrib.contenttypes',
        'django.contrib.sessions',
        'django.contrib.messages',
        'django.contrib.staticfiles',
    )
    
    # 補充：settings.py 常會使用到 DATABSES 參數
    # DATABSES 定義了使用的資料庫。預設會使用 db.sqlite3 這個 SQLite 資料庫。

啟動 Server

啟動 server 前，有在 DB 做異動時，例如新建一個 Model，就必須執行以下指令，目的是確保 code 與 database 的狀態。

  • 執行 python manage.py makemigrations 來看哪些地方有做更動(這邊還不會做更新)
  • 執行 python manage.py migrate migration 會把 database 調整到符合現在 code 的狀態，並記錄在 myapp/migrations/ 底下。
  • 執行 python manage.py runserver 就可以至 http://127.0.0.1:8000/ 看到畫面
  • 若想跟換 port 如 8080 則可執行 python manage.py runserver 8080

Usage

Django URL Dispatcher
  • Flask 寫法
  • Django 寫法
  • 把 URL 加在應用程式(APP)

    • 在 myapp 資料夾底下建立 urls.py。

    
    # myapp/urls.py
    from django.urls import path
    from . import views  # 引用這個資料夾中的 views 檔案
    urlpatterns = [
        # 表示 http://127.0.0.1:8000/myapp/overview 會進入 index 的 function 處理
        path('overview', views.index) 
    ]
  • 把 URL 加在 project 設定

    • 應用程式(APP)的網址定義完成後，接下來就要註冊到 Django 專案主程式(potrip)，讓它知道有 posts 這個應用程式(APP)的存在。

    
    from django.contrib import admin
    from django.urls import path, include
    
    urlpatterns = [
        path('admin/', admin.site.urls),
        path('myapp/', include('myapp.urls')) # 新增應用程式的網址
    ]
    
    # 透過 include 函式，即可將應用程式(APP)中的所有網址註冊到 Django 專案主程式(potrip)中
    # 當使用者存取 http://127.0.0.1:8000/myapp/ 開頭的網址時，
    # Django 就知道要去找 myapp 應用程式(APP)中的網址。

Django Model

這邊就像是建立一個 DB Schema。


    from django.db import models
    from django.utils import timezone

    class Post(models.Model):
        title = models.CharField(max_length=200) # 這是一個字串，有上限數字
        text = models.TextField() # 這是一個無上限的字串
        created_date = models.DateTimeField(default=timezone.now)
        published_date = models.DateTimeField(blank=True, null=True)

Django QuerySet

QuerySet 允許你從資料庫中讀取資料，也可以對資料做篩選或排序。打開終端機輸入 python manage.py shell，進入互動介面，再匯入 Law_report 就可以看到目前的資料。


    from law.models import Law_report
    Law_report.objects.all()
    Law_report.objects.filter(firstName = "Vaibhav").delete() # 刪除

Django Sessions
  • 存取 server 端資料
  • 像是 Cookie 或是 ID 等等

    from django.contrib.sessions.backends.db import SessionStore                       
    request.session['page'] = page
    page = request.session.get('page', True)

Django Administration
  • admin.py 主要用來註冊資料模型(Model)及客製化 Django Administration(管理員後台)的顯示方式。
  • 終端機輸入 python manage.py createsuperuser 來建立超級使用者。
  • 如果想要把 Django 專案中的應用程式(APP) 資料模型(Model)，加入到 Django Administration(管理員後台)，可以透過應用程式(APP)底下的 admin.py 檔案來達成。

        PROJECT_PATH + '/templates/',
    )
